## 1. 问题清单（产品维度，按优先级）

### 🔴 致命级（产品本身不成立 / 用户不会用第二次）

#### P1 — 写一条只有"意象寄送"一种主动作
- **现象**：用户进入"写一条"，只能从 8 个意象里选一个 → 生成诗意 → 投出。  
- **风险**：日常 80% 的时刻是"没什么事要说但想表示关心"——意象寄送的仪式感太重，会让用户觉得"今天没什么大事，不寄了"。频次=0，留存=0。
- **修复方向**：在"写一条"入口增加两种轻动作——
  - **每日心情**：3 秒选一个心情图标 → 自动生成一句话寄出
  - **简单问候**：8 个高频短句（吃饭了吗 / 路上小心 / 今天加油）一键投出

#### P2 — 收件箱缺"回应"动作，往返闭环只有一半
- **现象**：启读家书 → 看完 → 关闭。没有"回一句"按钮。
- **风险**：情感传递的核心是**双向往返**，不是单向投递。看完即结束 = 朋友圈点赞水平。
- **修复方向**：启读详情页底部固定"回一句"CTA → 直接进入简化版"写一条"，自动绑定 receiverId 为原发件人。

#### P3 — 父母端是"小号孩子端"，不是真父母端
- **现象**：父母端目前只是把字号变大 / 配色变暖，交互逻辑与孩子端完全一致。
- **风险**：真实老人不会"选意象"、不会"打字润色"。父母端必须有专属交互。
- **修复方向**：父母端首页改为两张大卡——
  - **「看到孩子今天」**：展示孩子最近一封心意 + "我看到了"按钮
  - **「我也说一句」**：6 个超大问候卡，点一下即发送

---

### 🟡 重要级（产品有但不够好）

#### P4 — 留存抓手缺失（被动触发）
- **现象**：纯主动 APP，没有"今天 TA 心情变了"" 节气提醒""每日一句"这种被动触发。
- **修复方向**：本轮不实现完整推送，但首页加"今日家庭气象"占位 + "明日是 XX 节气"的轻提醒。

#### P5 — 情感证据缺失（发件人不知道对方有没有看到）
- **现象**：寄出后只能看到"已寄出"，无法知道对方启读、是否有反应。
- **修复方向**：收件箱启读后自动写"opened"状态；发件人的"我寄出"列表显示"已被启" / "等待启读"；启读后可点一个"心动"图标，发件人能看到。

#### P6 — AI 润色"看不见"，最该秀的差异化被藏起来
- **现象**：用户写一句粗糙的话 → AI 润色 → 只看到结果。
- **风险**：评委不知道 AI 做了什么；用户也感受不到 AI 的价值。
- **修复方向**：写一条预览页改为"上下对照"——上半"你想说的"（原话/意象），下半"AI 帮你说的"（润色后），中间一条细线 + "AI 润色"小标签。

### Step 1 —【P2 + P5】收件箱启读详情抽屉 + 回一句 + 心动回执
- **动机**：P2 往返闭环只完成一半（启读即结束，无"回一句"）；P5 发件人不知道对方是否看到、是否有反应。
- **改动**：
  - [src/lib/types.ts](file:///h:/Master%20Work/VIVO/family-bridge/src/lib/types.ts)：DriftBottle 新增 `originalText`、`mode`、`reaction`、`reactedAt` 字段；新增 `DriftBottleMode`、`DriftBottleReaction` 类型。
  - [src/lib/db.ts](file:///h:/Master%20Work/VIVO/family-bridge/src/lib/db.ts)：新增 `updateDriftBottle` 通用更新函数。
  - [src/lib/store.ts](file:///h:/Master%20Work/VIVO/family-bridge/src/lib/store.ts)：新增 `reactToDriftBottle` 动作。
  - [src/app/inbox/page.tsx](file:///h:/Master%20Work/VIVO/family-bridge/src/app/inbox/page.tsx)：完整重写。
    - 启读变为打开底部抽屉（BottleDetailDrawer），展示诗 + 意象 + 发件人
    - 底部固定「回一句」CTA（跳转 bottle 页，自动绑定对方为接收人）
    - 「心动」按钮（可点一次，不可撤销）
    - 已读/我寄出列表也支持点击回看详情
    - polish 模式在详情抽屉展示"原话 → AI 润色后"对比
- **结果**：收 → 启读 → 心动 → 回一句，完整往返闭环。

### Step 2 —【P1 + P6】写一条三模式 + AI 润色对比
- **动机**：P1 只有"意象寄送"一种高门槛主动作，频次不足；P6 AI 润色过程不可见，差异化被隐藏。
- **改动**：
  - [src/app/bottle/page.tsx](file:///h:/Master%20Work/VIVO/family-bridge/src/app/bottle/page.tsx)：完整重写。
    - 顶部三模式 Tab：意象寄送 / 每日心情 / 一键问候
    - **意象寄送**：选意象 → 确认接收人 → 可选输入原话 → AI 润色 → 预览（带前后对比）→ 投出
    - **每日心情**：6 个心情卡片（平静/开心/累了/感恩/想你/骄傲），点一下直接寄出
    - **一键问候**：8 个高频短句（吃饭了吗/路上小心/…），点一下直接寄出
    - 支持从收件箱"回一句"进入的 reply 模式（标题显示"回复 · XX"）
    - P6 对比：预览页上"你想说的"（原话，删除线）+ ↓ + "AI 帮你说的"（润色后）
- **结果**：频次问题解决（3 种方式，最轻只要 1 次点击）；AI 价值被显性化（评委一眼看到润色前后对比）。

### Step 3 —【P3】父母端首页差异化
- **动机**：父母端之前是"小号孩子端"，交互逻辑完全一致，不符合真实老人/父母的交互习惯。
- **改动**：
  - [src/app/page.tsx](file:///h:/Master%20Work/VIVO/family-bridge/src/app/page.tsx)：父母端分支完全重写。
    - 两张大卡：「看到孩子今天」+「我也说一句」
    - 「看到孩子今天」：展示孩子最新一封未读心意 + "我看到了"按钮（启读后跳 inbox）
    - 「我也说一句」：6 个大号问候卡（路上小心/吃饭了吗/…），点一下直接发送，不用打字
    - 底部快捷入口：收件箱 + 桌面组件
    - 同时修复 `unopenedBottles` 的 `receiverId === "self"` 硬编码 → `receiverId === currentMember?.id`
  - 孩子端保持不变（心湖风格 + 九宫格）。
- **结果**：父母端 = 独立交互形态（接住孩子 + 一键回应），不是孩子端的缩小版。
